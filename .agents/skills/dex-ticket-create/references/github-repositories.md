# GitHub repositories

## elastic/kibana (default)

- **Visibility**: Public
- **Use for**: Bugs, feature requests, technical tasks, some epics.
- **When to choose**: Default choice for most tickets. Use this unless the content needs to be private.

## elastic/security-team

- **Visibility**: Private
- **Use for**: Product epics, workstreams, initiatives. Also anything confidential: bugs with customer information, security issues, internal infrastructure work, tickets related to SDHs.
- **When to choose**: When the ticket content shouldn't be public — customer data, security vulnerabilities, internal infra details, or high-level product planning items.
